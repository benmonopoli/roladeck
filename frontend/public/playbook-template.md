# Senior Backend Engineer — Hiring Guide

## Criteria

**Tier 1 — Must Have**
- Has shipped production APIs (REST or GraphQL) at significant scale
- Strong command of at least one statically-typed language (Go, Rust, Java, OCaml, etc.)
- Experience with relational databases: schema design, indexing, query optimisation
- Solid understanding of concurrency, I/O, and performance trade-offs

**Tier 2 — Differentiator**
- Has designed distributed systems (queues, caches, eventual consistency)
- Meaningful open-source contributions or side projects demonstrating depth
- Experience with observability: structured logging, distributed tracing, metrics

**Tier 3 — Rare Upside**
- Experience with compiler or language tooling
- Prior startup founding or early-engineer experience

## Red Flags
- Can only describe high-level architecture but struggles to explain implementation details
- No production debugging experience (logs, profiling, distributed tracing)
- Rewrites from scratch without understanding the existing system

### Seniority Signals
- **Mid → Senior:** Leads technical design of a subsystem end-to-end
- **Senior → Staff:** Influences cross-team architecture decisions
- **Staff → Principal:** Drives multi-year technical strategy across the org
